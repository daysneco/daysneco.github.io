# 每日 AI 简报 - 2026-06-19

## 今日最重要的8条 AI 新闻

> 说明：过去 24 小时内，Transformer 论文共同作者 Noam Shazeer 跳槽 OpenAI、Google Gemini CLI 正式下线、xAI 发布视频模型 1.5 版等值得关注；广告软文和重复转载已剔除。

### 1. Transformer 论文共同作者 Noam Shazeer 离开 Google Gemini，加入 OpenAI
- 来源：CNBC / Reuters / Bloomberg
- 链接：https://www.cnbc.com/2026/06/18/google-gemini-co-lead-noam-shazeer-leaves-for-openai.html
- 这件事是什么：6 月 17–18 日，Google Gemini 联合负责人、工程副总裁 Noam Shazeer 在 X 上宣布离职，加入 OpenAI。他是 2017 年《Attention Is All You Need》论文共同作者，2024 年 Google 花约 27 亿美元把他从 Character.AI 挖回，不到两年又跳槽。
- 为什么重要：Transformer 架构的奠基人之一从 Google 核心 AI 团队流向最大竞争对手，对 Gemini 研发是重大打击，也帮 OpenAI 在 IPO 前补强了顶尖人才。
- 对我有什么影响：短期内模型产品不会立刻变，但说明 OpenAI 下一代架构可能更强。如果你重度依赖 Gemini，可关注后续团队变动是否影响产品节奏。

### 2. Google Gemini CLI 今日正式下线，个人用户被迫迁移 Antigravity CLI
- 来源：Google Developers Blog / TechTimes
- 链接：https://developers.googleblog.com/en/an-important-update-transitioning-gemini-cli-to-antigravity-cli/
- 这件事是什么：6 月 18 日，Google 按计划停止向免费、AI Pro、Ultra 及个人版 Gemini Code Assist 用户提供服务。所有调用 `gemini` 命令的脚本、CI/CD 流水线当场失效，需改用闭源的 Go 版 Antigravity CLI（命令 `agy`）。企业付费版不受影响。
- 为什么重要：这是 Google 把开源 CLI 收进付费/企业体系的标志性动作。10 万+ GitHub 社区贡献者搭建的工具链一夜断裂，多智能体异步编排成了新卖点，但功能尚未完全对齐。
- 对我有什么影响：如果你用 Gemini CLI 做自动化开发，今天就得检查脚本是否报错，尽快迁移到 `agy` 或换 Claude Code、Aider 等替代方案。企业用户可继续用 Gemini CLI，不必急着改。

### 3. xAI 发布 Grok Imagine Video 1.5，6 秒 720p 视频约 25 秒生成
- 来源：Winbuzzer
- 链接：https://winbuzzer.com/2026/06/18/xai-speeds-up-grok-imagine-video-15-for-ai-video-creators-xcxwbn/
- 这件事是什么：6 月 18 日，xAI 正式发布 Grok Imagine Video 1.5，通过 API 和 App 开放。Fast 模式可在约 25 秒内生成 6 秒 720p 视频，并同步生成配音和环境音。还新增项目管理、多 Agent 协作和素材库搜索等创作工具。
- 为什么重要：AI 视频生成竞争进入"速度 + 音画同步"阶段。xAI 把生成时间从 40 秒压到 25 秒，但最高仍只有 720p，和 Google Veo、Runway 的 1080p 还有差距。
- 对我有什么影响：做短视频、广告创意的团队可以试用新 API 加速出片。如果你需要 1080p 交付给甲方，目前还得等 xAI 升级或继续用竞品。

### 4. 马斯克公开承诺：年底前用 AI 做出完整长片电影
- 来源：Singularity.Kiwi / xAI 官方
- 链接：https://singularity.kiwi/xai-full-movies-individualised-content-musk-2026/
- 这件事是什么：6 月 17 日，xAI 发布了一段 2 分 06 秒的 AI 电影预告片。马斯克在评论区直接回复"年底前出完整电影"。OpenAI Sora、Google Veo 3、Runway 目前最长只做到 8–10 秒片段，没人公开承诺全长作品。
- 为什么重要：这是行业第一次有人把"AI 做电影"从演示片拉到有截止日期的公开承诺。xAI 的打法可能是"每人一部不同电影"的个性化内容，而不只是复制好莱坞模式。
- 对我有什么影响：对普通用户暂无可用产品，但说明 AI 视频能力在快速逼近实用门槛。内容创作者可关注 xAI 后续开放情况，影视从业者需提前思考 AI 对制作流程的冲击。

### 5. Anthropic 推出 Claude Code Artifacts，团队可共享实时交互看板
- 来源：VentureBeat
- 链接：https://venturebeat.com/data/anthropics-claude-code-artifacts-update-brings-live-shared-dashboards-and-interactive-workspaces-to-enterprises
- 这件事是什么：6 月 18 日，Anthropic 为 Claude Team 和 Enterprise 用户上线 Artifacts 功能：Claude Code 会话产出的代码、数据可一键变成可分享的交互式网页，比如实时数据看板、内部工具原型，同事打开链接就能用，无需部署后端。
- 为什么重要：这和 OpenAI Codex 两周前推出的 Sites 功能正面交锋，但 Anthropic 刻意不做后端托管，降低企业安全和合规顾虑。AI 编程工具正从"写代码"进化到"交付可用产品"。
- 对我有什么影响：如果你公司有 Claude Enterprise 订阅，现在可以让 AI 直接产出团队能用的内部页面，省去前端开发和部署环节。个人用户暂时用不了，可关注后续是否下放。

### 6. 摩根大通切断香港员工对 Anthropic Claude 的访问
- 来源：The Next Web / Financial Times
- 链接：https://thenextweb.com/news/jpmorgan-cuts-off-anthropic-access-for-hong-kong-staff
- 这件事是什么：据《金融时报》报道，摩根大通已将 Anthropic 的 Claude 从香港员工可用的 AI 工具列表中移除，原因是许可协议条款措辞问题，而非模型性能。今年 4 月高盛已对香港员工做过同样限制。背景是美国商务部要求 Anthropic 限制先进模型对外出口。
- 为什么重要：Fable 5 禁令的余波正在金融圈扩散。华尔街大行因合规顾虑主动收缩 Claude 在特定地区的使用，AI 工具的"地缘政治风险"从理论变成实操。
- 对我有什么影响：如果你在金融、跨境业务中使用 Claude，要检查公司合规政策是否收紧。香港及敏感地区用户应提前准备替代模型，别把关键流程绑在可能被切断的工具上。

### 7. 欧盟网络安全局在美方出口禁令下与 Anthropic 会面
- 来源：The Next Web
- 链接：https://thenextweb.com/news/eu-cybersecurity-agency-to-meet-anthropic-under-a-us-export-cloud
- 这件事是什么：6 月 19 日，欧盟网络安全局 ENISA 按计划与 Anthropic 会面，讨论前沿模型早期访问合作（Project Glasswing）。但美国商务部刚下令 Anthropic 暂停向外国公民提供最先进模型，这次"给欧洲开绿灯"的谈判变得极其尴尬。
- 为什么重要：这是"美国 AI 出口管制"与"欧洲数字主权"的直接碰撞。Anthropic 一边想跟欧洲监管机构合作，一边要遵守本国政府的禁售令，两难局面可能变成所有美国 AI 公司的常态。
- 对我有什么影响：欧洲用户和企业获取美国前沿模型的渠道可能进一步收窄。如果你在欧盟做 AI 业务，要关注"可信伙伴"机制是否落地，以及本地替代方案（Mistral 等）的可用性。

### 8. Cast AI 将 MiniMax M3 设为 Kimchi 编码平台默认模型
- 来源：IT Brief
- 链接：https://itbrief.news/story/cast-ai-adds-minimax-m3-to-kimchi-coding-as-default-model
- 这件事是什么：6 月 18 日，云成本优化公司 Cast AI 把国产开源模型 MiniMax M3 接入 Kimchi Coding 自主编程平台，作为默认 Builder 模型。M3 在 SWE-bench Pro 真实 GitHub 问题测试中得分 59%，支持百万 token 上下文，可自托管在 AWS、Azure 或本地。
- 为什么重要：这是国产开源模型首次被海外企业级编码 Agent 平台设为默认，说明 M3 在代码能力和成本上已具备国际竞争力，不再只是"中国版替代品"。
- 对我有什么影响：如果你在找比 Claude API 更便宜的编程 AI，可以试试 Kimchi + M3 组合，支持私有化部署，数据不出内网。开源权重也意味着可以自己微调适配业务场景。
